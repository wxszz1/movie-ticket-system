# 电影推荐与票务系统 — 期末综合报告

**课程：** JavaScript 相关技术
**项目名称：** 电影推荐与票务系统
**完成日期：** 2026年6月29日

## 一、需求分析

### 1.1 项目背景

随着互联网技术的快速发展，在线电影票务平台已成为人们观影消费的主要渠道。猫眼电影、淘票票等应用为用户提供了便捷的购票体验。本项目旨在模拟构建一个功能完整的电影推荐与票务系统，涵盖用户管理、影片浏览、选座购票、推荐系统、营销社交和后台管理六大核心模块。

### 1.2 功能需求

#### 用户管理模块
- 手机号/邮箱注册与密码登录
- 修改密码、忘记密码
- 个人资料管理（昵称、头像）
- 用户分级：普通用户 / VIP / 超级VIP
- 积分体系：购票 +10 分、评价 +5 分、每日签到 +2 分
- 会员等级权益（折扣、优先选座）

#### 影院与影片管理模块
- 影片列表（分页、筛选、搜索）
- 影片详情（海报、预告片、演职员表、评分）
- 影院信息展示（地址、设施标签）
- 影厅管理（IMAX/杜比/普通、座位布局）
- 排片管理（场次、价格策略）
- 用户评分与短评
- 想看/看过标记

#### 选座购票模块（核心）
- 选座流程：影片 → 影院 → 场次 → 座位
- SVG 交互式座位图（可选/已售/锁定/故障状态）
- 15 分钟支付倒计时
- 订单创建、模拟支付、取消、退款
- 取票码生成

#### 推荐系统模块
- 首页个性化推荐（基于用户历史标签匹配）
- 相似影片推荐（同类型/同导演）
- 热门推荐（近期票房排行）
- 冷启动策略（新用户推荐热门 + 随机）

#### 营销与社交模块
- 优惠券系统（满减券、折扣券、代金券）
- 评分与短评
- 想看/看过列表
- 观影成就系统

#### 后台管理模块
- 数据看板（总用户数、总票房、今日订单、用户增长曲线、热门影片排行）
- 影片信息编辑
- 排片管理（新增场次）
- 用户管理（角色权限设置）
- 订单管理（查询、退款审核）

### 1.3 非功能需求

| 需求类型 | 具体要求 |
|----------|----------|
| 性能 | 页面首屏加载 < 2秒，API 响应 < 500ms |
| 可用性 | 响应式设计，支持 PC 和移动端 |
| 安全性 | JWT 认证、密码 bcrypt 加密、权限中间件 |
| 可维护性 | 模块化代码结构、RESTful API 设计 |
| 数据完整性 | SQLite 外键约束、事务操作 |

## 二、系统设计

### 2.1 技术选型

| 层级 | 技术 | 选型理由 |
|------|------|----------|
| 前端 | 原生 JavaScript + ES Modules | 课程要求，无框架依赖，深入理解 JS 基础 |
| 路由 | Hash 路由 | 单页应用无需后端配合，实现简单 |
| 样式 | CSS3 (Flexbox + Grid) | 响应式布局，无需预处理器 |
| 后端 | Node.js + Express 5 | 异步 I/O，与前端统一语言 |
| 数据库 | SQLite (better-sqlite3) | 零配置、单文件存储、适合演示 |
| 认证 | JWT (jsonwebtoken) | 无状态认证，前后端分离友好 |
| 密码 | bcryptjs | 单向哈希，安全性高 |
| 座位图 | SVG 动态渲染 | 矢量图不失真，支持交互事件 |

### 3.2 核心功能实现

#### 3.2.1 用户认证系统

采用 JWT (JSON Web Token) 实现无状态认证：

```
登录流程：
用户提交账号密码 → 后端验证 → 生成 JWT Token → 返回前端
→ 前端存储 Token 到 localStorage → 后续请求携带 Authorization 头
→ 中间件验证 Token → 通过则放行，否则返回 401
```

密码使用 bcryptjs 进行单向哈希加密，数据库中不存储明文密码。权限中间件通过检查 `user.role` 字段区分普通用户和管理员。

#### 3.2.2 SVG 座位图

选座功能是系统的核心交互，采用 SVG 动态渲染：

```javascript
// 座位渲染核心逻辑
function renderSeatMap(container, seats, layout) {
  const svg = document.createElementNS('http://www.w3.org/2000/svg', 'svg');
  const seatSize = 30, gap = 4;

  seats.forEach(seat => {
    const rect = document.createElementNS('http://www.w3.org/2000/svg', 'rect');
    rect.setAttribute('x', (seat.col - 1) * (seatSize + gap));
    rect.setAttribute('y', (seat.row - 1) * (seatSize + gap));
    rect.setAttribute('width', seatSize);
    rect.setAttribute('height', seatSize);
    rect.setAttribute('rx', 4);
    rect.setAttribute('fill', getSeatColor(seat.status));
    rect.addEventListener('click', () => toggleSeat(seat));
    svg.appendChild(rect);
  });
  container.appendChild(svg);
}
```

颜色编码：绿色 = 可选、灰色 = 已售、黄色 = 选中、红色 = 故障。

座位锁定采用 15 分钟超时机制，用户选座后创建 pending 订单，超时自动释放。

#### 3.2.3 推荐算法

采用基于内容的推荐 + 热门推荐的混合策略：

1. **基于内容推荐**：提取用户历史观看影片的 genre 标签，匹配同标签影片
2. **热门推荐**：按评分和购票数加权排序
3. **冷启动策略**：新用户无历史数据时，直接展示热门 + 随机推荐

```javascript
// 推荐核心逻辑
GET /api/recommend/home
1. 查询用户想看/看过的电影 genre
2. 用 LIKE 匹配同 genre 的其他电影
3. 按评分排序返回
4. 新用户 fallback 到热门推荐
```

#### 3.2.4 首页设计

首页采用 Hero Banner + 电影网格的布局：
- 顶部 Hero：热门电影大图推荐，模糊背景 + 海报 + 评分 + 简介 + 购票按钮
- 电影卡片：渐变遮罩、hover 放大、右上角评分徽章
- 三区分栏：热门推荐 / 正在热映 / 即将上映

### 3.3 遇到的问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| Express 5 路由不兼容 | `app.get('*', ...)` 在 Express 5 中无效 | 改为 `app.get('/{*splat}', ...)` |
| 数据库表未自动创建 | init-tables.js 未被 index.js 引用 | 在 index.js 启动时 require |
| SQLite 自增 ID 不重置 | DELETE 后 autoincrement 计数器不归零 | 同时删除 sqlite_sequence 表记录 |
| 密码验证失败 | seed.js 用 SHA-256 而非 bcrypt | 改用 bcrypt.hashSync |
| 座位查询 SQL 报错 | 缺少跨表 JOIN | 补充 schedules/halls/movies/cinemas JOIN |
| 推荐 API LIKE 模式错误 | 参数中包含多余的双引号 | 移除字面量引号 |
| 缺少 POST /user-movies | 前端调用但后端未实现 | 新增 upsert 逻辑的 POST 端点 |
| 前端代码模式不匹配 | 计划中的 API 与实际实现不一致 | 子代理适配实际代码模式 |

## 四、总结与反思

### 4.1 项目成果

本项目成功实现了一个功能较为完整的电影推荐与票务系统，包含以下核心功能：

- **用户系统**：注册登录、JWT 认证、资料管理、积分签到
- **影片浏览**：海报展示、详情信息、评分评价、想看/看过
- **选座购票**：SVG 交互式座位图、座位锁定、订单创建、模拟支付
- **推荐系统**：个性化推荐、相似影片、热门排行
- **后台管理**：数据看板、影片/排片/订单/用户管理
- **营销功能**：优惠券系统

技术栈选择合理，原生 JS 前端 + Node.js 后端 + SQLite 数据库的组合降低了环境配置复杂度，适合课程项目的演示和答辩。

### 4.2 技术收获

1. **全栈开发能力**：从零搭建了完整的前后端系统，理解了 RESTful API 设计、JWT 认证、数据库建模等核心概念
2. **模块化思维**：将 17 个独立任务拆解并逐个实现，学会了如何将复杂系统分解为可管理的单元
3. **问题排查能力**：在开发过程中遇到了 Express 5 兼容性、SQL 查询错误、前端路由等多个问题，通过日志分析和逐步调试解决
4. **Git 版本管理**：全程使用 Git 进行版本控制，理解了提交规范和分支管理的重要性

### 4.3 不足与改进方向

| 不足之处 | 改进方向 |
|----------|----------|
| 忘记密码功能未实现 | 接入短信验证码或邮箱验证 |
| 头像上传未实现 | 使用 multer 处理文件上传，存储到本地或 OSS |
| 上座率分析未实现 | 增加统计维度，按场次/影院/时段分析 |
| 广告位管理未实现 | 增加广告位 CRUD 和前端轮播展示 |
| 系统日志未实现 | 使用 winston/morgan 记录操作日志 |
| 推荐算法较简单 | 引入协同过滤或机器学习模型 |
| 无实时通信 | 可引入 WebSocket 实现实时座位状态同步 |
| 无自动化测试 | 补充单元测试和集成测试 |

### 4.4 个人感悟

作为一人独立完成的全栈项目，从需求分析、系统设计到编码实现、测试部署，整个过程让我对 Web 开发有了更深入的理解。特别是在遇到问题时，学会了如何通过查阅文档、分析日志、逐步排查来定位和解决问题。

项目开发中最大的挑战在于时间管理和功能取舍。8 天的开发周期内要完成 6 大模块，必须合理安排优先级，确保核心功能（选座购票）的质量，同时对非核心功能进行适当简化。

总的来说，本项目达到了预期目标，实现了一个可演示、可交互的电影票务系统，也让我在 JavaScript 全栈开发方面积累了宝贵的实践经验。
